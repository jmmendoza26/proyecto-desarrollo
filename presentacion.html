Introducción

Objetivos del Proyecto

Problema

Reconocimiento automático de frutas y vegetales a partir de imágenes. Aplicaciones: inventario automatizado, nutrición personal, control de calidad en cadenas de suministro.

Objetivo General

Desarrollar y comparar transfer learning con fine-tuning parcial de ResNet18 y una CNN diseñada desde cero, evaluando el desempeño en condiciones controladas y en fotografías reales.

5 Clases de Clasificación

Apple Banana Tomato Potato Onion

Objetivos Específicos

  • Implementar un pipeline completo en PyTorch con funciones reutilizables, desde adquisición del dataset hasta evaluación con fotografías reales.
  • Aplicar transfer learning con fine-tuning parcial sobre ResNet18, descongelando layer4 y la capa clasificadora fc.
  • Entrenar una CNN de 4 bloques convolucionales desde cero bajo las mismas condiciones experimentales.
  • Comparar accuracy, F1-score y desempeño con fotografías reales para cuantificar el impacto del conocimiento preentrenado de ImageNet.
  • Evaluar generalización ante la brecha de dominio (domain shift) entre imágenes controladas y fotografías reales.

Setup del Modelo y Configuración de Entrenamiento

resnet_setup.py — Fine-tuning parcial + Optimizer con tasas diferenciadas 
# 1. Cargar ResNet18 con pesos de ImageNet
resnet = models.resnet18(weights=RESNET_WEIGHTS)

# 2. Congelar TODAS las capas
for param in resnet.parameters():
    param.requires_grad = False

# 3. Descongelar layer4 para fine-tuning de alto nivel
for param in resnet.layer4.parameters():
    param.requires_grad = True

# 4. Reemplazar clasificador: 1000 clases → 5 clases
resnet.fc = nn.Linear(resnet.fc.in_features, len(GENERAL_CLASSES))
resnet = resnet.to(device)

# Verificación: 11,179,077 total | 8,396,293 entrenables (75%)
trainable = sum(p.numel() for p in resnet.parameters() if p.requires_grad)
total     = sum(p.numel() for p in resnet.parameters())

# 5. Optimizer con tasas de aprendizaje DIFERENCIADAS por grupo
optimizer_resnet = optim.Adam([
    {"params": resnet.layer4.parameters(), "lr": 1e-4},  # conservador
    {"params": resnet.fc.parameters(),     "lr": 1e-3},  # estándar
])

# 6. Scheduler: reduce lr a la mitad cada 5 épocas
scheduler_resnet = optim.lr_scheduler.StepLR(
    optimizer_resnet, step_size=5, gamma=0.5
)

# 7. Entrenamiento — 20 épocas
history_resnet = train_model(
    model=resnet, train_loader=train_loader,
    val_loader=val_loader, criterion=nn.CrossEntropyLoss(),
    optimizer=optimizer_resnet, epochs=20,
    scheduler=scheduler_resnet,
)

Parte 2 · CNN desde Cero

Arquitectura — FruitVegetableCNN

model.py · FruitVegetableCNN 
class FruitVegetableCNN(nn.Module):
    def __init__(self, num_classes=5):
        super().__init__()

        def conv_block(in_ch, out_ch):
            return nn.Sequential(
                nn.Conv2d(in_ch, out_ch, kernel_size=3, padding=1),
                nn.BatchNorm2d(out_ch),
                nn.ReLU(inplace=True),
                nn.MaxPool2d(2),
            )

        # 4 bloques: filtros aumentan 3→32→64→128→256
        self.features = nn.Sequential(
            conv_block(3,   32),
            conv_block(32,  64),
            conv_block(64,  128),
            conv_block(128, 256),
        )

        # Elimina dependencia del tamaño de entrada
        self.gap = nn.AdaptiveAvgPool2d(1)

        self.classifier = nn.Sequential(
            nn.Flatten(),
            nn.Linear(256, 128),
            nn.ReLU(inplace=True),
            nn.Dropout(0.4),
            nn.Linear(128, num_classes),
        )

    def forward(self, x):
        x = self.features(x)
        x = self.gap(x)
        x = self.classifier(x)
        return x

Conclusiones

Limitaciones y Conclusiones

Limitaciones

  • Domain shift persistente: Potato obtuvo 0% en fotos reales en ambos modelos. 10 imgs propias/clase son insuficientes.
  • Validación optimista: el conjunto de validación usa solo imágenes Fruits-360 (fondo blanco), inflando las métricas.
  • Inestabilidad CNN: fluctuaciones de ±39 pp indican que 450 imgs son insuficientes para 422K parámetros desde cero.
  • Muestra real pequeña: 40 fotos con distribución desigual (12 Apple vs 6 Potato) sesga el accuracy total.

Mejoras Futuras

  • Ampliar dataset propio a 50–100 imgs reales/clase (énfasis en Potato y Onion).
  • Implementar early stopping basado en val accuracy para evitar retrocesos.
  • Construir conjunto de validación real balanceado (≥20 imgs/clase no controladas).

Conclusiones Clave

  • Transfer learning logró 99% val accuracy en época 3: el conocimiento preentrenado de ImageNet acelera radicalmente la convergencia.
  • La CNN desde cero, con 26× menos parámetros, fue competitiva en dominio controlado (76%) pero no generalizó a fotos reales (48% vs 70%).
  • El domain shift es el principal obstáculo: las imágenes con fondo blanco de Fruits-360 difieren fundamentalmente de fotografías reales.
  • Las funciones reutilizables garantizaron condiciones idénticas en ambos modelos: la comparación es estrictamente válida.
